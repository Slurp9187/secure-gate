</span></div><details class="toggle top-doc" open><summary class="hideme"><span>Expand description</span></summary><div class="docblock"><p>Zero-cost secure wrappers for secrets — <a href="struct.Dynamic.html" title="struct secure_gate::Dynamic"><code>Dynamic&lt;T&gt;</code></a> for heap-allocated variable-length data,
<a href="struct.Fixed.html" title="struct secure_gate::Fixed"><code>Fixed&lt;T&gt;</code></a> for stack-allocated fixed-size data.</p>
<p>This crate provides explicit, guarded wrappers for sensitive values (e.g. keys, tokens, ciphertexts)
with controlled exposure via <code>.expose_secret()</code> / <code>.expose_secret_mut()</code>. No accidental leaks via
<code>Deref</code>, <code>AsRef</code>, or implicit conversions.</p>
<p>See <a href="../README.md">README.md</a> for usage examples, feature overview, and macros for custom aliases.</p>
<h3 id="equality-options"><a class="doc-anchor" href="#equality-options">§</a>Equality Options</h3>
<p><a href="trait.ConstantTimeEq.html" title="trait secure_gate::ConstantTimeEq"><code>ConstantTimeEq</code></a> (via <code>ct-eq</code> feature): Direct byte-by-byte constant-time comparison using <code>subtle</code>.
Best for small/fixed-size secrets (&lt; ~256–512 bytes) where speed matters most.</p>
<p><a href="trait.ConstantTimeEqExt.html" title="trait secure_gate::ConstantTimeEqExt"><code>ConstantTimeEqExt</code></a> (via <code>ct-eq-hash</code> feature): BLAKE3 hash -&gt; constant-time compare on fixed 32-byte digest.
Faster for large/variable secrets (e.g. ML-KEM ciphertexts ~1–1.5 KiB, ML-DSA signatures ~2–4 KiB),
with length hiding and optional keyed mode (<code>rand</code> for per-process random key).</p>
<p>See the ConstantTimeEqExt trait documentation for performance numbers, security properties (probabilistic, timing-safe), and guidance on when to choose each (or hybrid).</p>
